## Key features

### Variants

Inspired by [Stitches](https://stitches.dev) they are a great way to create a consistent design system, so we created **UnoCSS Variants** to bring them to UnoCSS.

::docs-uv-component
---
as: button
size: sm
color: secondary
uv:
  base: 'font-medium bg-blue-500 text-white rounded-full active:opacity-80'
  variants:
    color:
      primary: 'bg-blue-500 text-white'
      secondary: 'bg-purple-500 text-white
    size:
      sm: 'text-sm'
      md: 'text-base'
      lg: 'px-4 py-3 text-lg'
  defaultVariants:
    size: 'md'
    color: 'primary'
  compoundVariants:
    - size: ['sm', 'md'],
      class: 'px-3 py-1'
---
Click me
::

<br />
<br />

```vue
<script setup>
import { uv } from 'unocss-variants';

const button = uv({
  base: 'font-medium bg-blue-500 text-white rounded-full active:opacity-80',
  variants: {
    color: {
      primary: 'bg-blue-500 text-white',
      secondary: 'bg-purple-500 text-white'
    },
    size: {
      sm: 'text-sm',
      md: 'text-base',
      lg: 'px-4 py-3 text-lg'
    }
  },
  compoundVariants: [
    {
      size: ['sm', 'md'],
      class: 'px-3 py-1'
    }
  ],
  defaultVariants: {
    size: 'md',
    color: 'primary'
  }
});
</script>

<template>
  <button :class="button({ size: 'sm', color: 'secondary' })">
    Click me
  </button>
</template>
```

To learn more about variants, check the [variants](/unocss-variants/variants) page.

### Overrides

**UnoCSS Variants** provides a `class` / `className` prop for overriding classes on any component.

<br />

<button class="rounded-full bg-pink-500 px-3 py-1 font-semibold color-white hover:bg-pink-500 active:opacity-80 mb-4">
  Button
</button>

```ts {17}
import { uv } from 'unocss-variants';

const button = uv({
  base: 'font-semibold text-white py-1 px-3 rounded-full active:opacity-80',
  variants: {
    color: {
      primary: 'bg-blue-500 hover:bg-blue-700',
      secondary: 'bg-purple-500 hover:bg-purple-700',
      success: 'bg-green-500 hover:bg-green-700',
      error: 'bg-red-500 hover:bg-red-700'
    }
  }
});

button({
  color: 'secondary',
  class: 'bg-pink-500 hover:bg-pink-500' // overrides the color variant
});

/**
 * Result:
 * font-semibold text-white py-1 px-3 rounded-full active:opacity-80 bg-pink-500 hover:bg-pink-500
 */
```

To learn more the overrides, check out this [page](/unocss-variants/overriding-styles).

### Components composition

**UnoCSS Variants** allows you to compose components using the `extend` parameter. It automatically merges the `classes`, `slots`, `variants`, `defaultVariants` and `compoundVariants` of the extended component.

<br />

<div class="flex gap-3 mb-4">
  <button class="rounded-full bg-zinc-100 px-3 py-1 font-semibold hover:bg-zinc-200 active:opacity-80 dark:bg-zinc-800 dark:text-white dark:hover:bg-zinc-800">
    Button
  </button>
  <button class="rounded-lg bg-blue-500 px-3 py-1 text-sm font-semibold uppercase tracking-wider text-white shadow-lg shadow-blue-500/50 hover:bg-blue-600 active:opacity-80 dark:bg-blue-500 dark:text-white dark:hover:bg-blue-600">
    Buy button
  </button>
</div>

```vue {20}
<script lang="ts" setup>
import { uv } from 'unocss-variants';

const baseButton = uv({
  base: [
    'font-semibold',
    'dark:text-white',
    'py-1',
    'px-3',
    'rounded-full',
    'active:opacity-80',
    'bg-zinc-100',
    'hover:bg-zinc-200',
    'dark:bg-zinc-800',
    'dark:hover:bg-zinc-800',
  ],
});

const buyButton = uv({
  extend: baseButton,
  base: [
    'text-sm',
    'text-white',
    'rounded-lg',
    'shadow-lg',
    'uppercase',
    'tracking-wider',
    'uno-layer-uv:bg-blue-500',
    'uno-layer-uv:hover:bg-blue-600',
    'shadow-blue-500/50',
    'uno-layer-uv:dark:bg-blue-500',
    'uno-layer-uv:dark:hover:bg-blue-600',
  ],
});
</script>

<template>
  <div class="flex gap-3">
    <button :class="baseButton()">
      Button
    </button>
    <button :class="buyButton()">
      Buy button
    </button>
  </div>
</template>

/**
 * buyButton();
 *
 * Result:
 * font-semibold dark:text-white py-1 px-3 rounded-full active:opacity-80 bg-zinc-100 hover:bg-zinc-200 dark:bg-zinc-800 dark:hover:bg-zinc-800 text-sm text-white rounded-lg shadow-lg uppercase tracking-wider uno-layer-uv:bg-blue-500 uno-layer-uv:hover:bg-blue-600 shadow-blue-500/50 uno-layer-uv:dark:bg-blue-500 uno-layer-uv:dark:hover:bg-blue-600
 */
```

To learn more about **Components composition**, check out this [page](/unocss-variants/composing-components).

### Developer experience

**UnoCSS Variants** is built with developer experience in mind, it provides a great autocomplete experience thanks to the fully-typed API, so when using TypeScript, `slots`, `values`, and `breakpoints` will be auto-completed for you.

### Conflict resolution

**UnoCSS Variants** does not include built-in conflict resolution, so you must manage class conflicts manually. While tools like `tailwind-merge` provide automatic resolution, relying on them can obscure which classes are actually applied, potentially leading to unexpected results. Since we want to keep the philosophy of CSS which is "cascading" and "specificity", we believe that managing conflicts manually is a better approach.
